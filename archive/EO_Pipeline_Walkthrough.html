Core Philosophy

Configuration Is Law

Every tag formula, bound, constraint, and objective lives in the feature file (Excel / database). The Python code is a dumb executor — it reads config and runs it. Nothing is hardcoded.

  • Change a boiler constraint? Update the DB row. No redeploy.
  • Add a new plant? Load a new case_id. Same code runs it.
  • All 41 tables loaded at startup via load_config_from_excel()
Nodes Are Atoms

Every function does exactly one thing. 10–30 lines, no side effects, pure inputs → outputs. You can unit-test each node in isolation and swap implementations without touching anything else.

  • create_pi_connection ≠ fetch_pi_data ≠ standardize_columns
  • build_variables ≠ build_constraints ≠ build_objective
  • DAG enforces execution order — no manual sequencing
DAG-Enforced Order

Inferred tags depend on each other. We never manually specify a calculation order. Instead, we parse every formula for [tag] references, build a NetworkX directed graph, and topological sort it.

  • 3,050 nodes, 6,318 edges built automatically from formulas
  • 5 circular dependency groups detected and flagged for fsolve
  • nx.is_directed_acyclic_graph() validates before execution
Pipelines Are Reusable

The same 8-stage pipeline runs for every plant: ETH, EOEG1, EOEG2, EOEG3, LAO. The model_id parameter filters all config tables to the right plant. One codebase, many plants.

  • load_config_from_db(case_id) is the only thing that changes per run
  • Sub-models, bounds, and constraints all resolve from filtered tables
  • Simulation mode lets you run without a live PI historian
💡
The Big Idea: The feature file (EO_UN_FF_Rev_14.xlsx) is the program. The Python code is just the runtime that interprets it. A process engineer can change optimization behaviour — add a furnace, tighten a bound, rewrite a formula — without touching a single line of Python.

def fetch_pi_data_simulated(
    pi_connection, tag_config, pipeline_macros,
    reference_timestamp=None

        if "temperature" in name_lower:

Inferred Tag Engine (DAG)
rm_block: data_enrichment_inferred_calculation
2,272 tags computed ⏱ 10.9s
▶
What happened

This is the most complex stage. 2,272 inferred tags (from the inferred_details table) have formula expressions that reference other tags. They must be evaluated in the right order — if tag B's formula depends on tag A, A must be computed first.

We never manually specify an order. Instead:

1Parse [tag] refsregex on every formula
2Build NetworkX DiGraph3,050 nodes · 6,318 edges
3Detect circular SCCs5 groups found
4Topological sortsafe linear order
5df.eval() each formularow-by-row substitution
6fsolve circular blocksfixed-point iteration

Circular groups detected (BLR boilers):

⟳BLR_1_HPS_Gen ↔ Spec_En_Cons_BLR_1 ⟳BLR_2_HPS_Gen ↔ Spec_En_Cons_BLR_2 ⟳BLR_3_HPS_Gen ↔ Spec_En_Cons_BLR_3 ⟳BLR_4_HPS_Gen ↔ Spec_En_Cons_BLR_4 ⟳BLR_5_HPS_Gen ↔ Spec_En_Cons_BLR_5

Each boiler's HPS generation depends on specific energy consumption, which itself depends on HPS generation — a classic simultaneous equation. scipy.optimize.fsolve treats them as unknowns x and finds x where f(x) = x.

# 1. Extract [tag] refs from formula strings
TAG_REF_PATTERN = re.compile(r"\[([^\[\]]+)\]")

def build_dependency_dag(formulas, known_inputs):
    dag = nx.DiGraph()
    for tag, formula in formulas.items():
        refs = extract_tag_refs(formula)
        for ref in refs:
            dag.add_edge(ref, tag) # ref → tag
    return dag

# 2. Topological sort — safe evaluation order
def topological_sort_tags(dag):
    sccs = nx.strongly_connected_components(dag)
    circular = [s for s in sccs if len(s) > 1]
    # remove circular nodes, topo-sort the rest
    clean_dag = dag.copy()
    clean_dag.remove_nodes_from(circular_tags)
    return nx.topological_sort(clean_dag), circular

# 3. Solve circular blocks with scipy fsolve
def solve_circular_block(df, circular_tags, formulas):
    def residuals(x):
        local_ctx = {**context, **dict(zip(tag_list, x))}
        return [eval_formula(formulas[t], local_ctx) - x[i]
                for i, t in enumerate(tag_list)]
    solution = fsolve(residuals, initial_guess)
    return solution

def run_sub_model_pipeline(df, sub_model,
                              model_registry=None):
    sorted_models = sort_sub_models(sub_model)

    for _, row in sorted_models.iterrows():
        model_type = row["sub_model_type"]

        if model_type == "equation":
            df = execute_equation_model(df, row)

        elif model_type in ("regression", "lgbm"):
            df = execute_ml_model(df, row,
                                    model_registry)
        elif model_type == "iterative":
            # handled by inferred engine
            pass

    return df

# Each equation model is just:
def execute_equation_model(df, sub_model_row):
    tag   = sub_model_row["sub_model_name"]
    expr  = sub_model_row["sub_model_expression"]
    df[tag] = [eval_formula(expr, row)
               for _, row in df.iterrows()]
    return df

Optimizer (DE + SLSQP Fallback)
rm_block: optimizer_model_main
141 vars optimized ⏱ 7.8s
▶
What happened

The optimizer searched over 141 decision variables to find the combination that minimizes Objective_2 (Total Operating Cost).

Two optimizers ran in sequence:

1Differential Evolution68,973 evaluations · 30 iters
2SLSQP fallbackgradient-based refinement
✓Validate solution0 bound violations
✓Build optimum DataFrame141 *_optimum columns added
🔍
Why two optimizers? Differential Evolution is a global, gradient-free method — good for the non-convex energy surface of furnaces. But with 141 variables and only 30 iterations (demo budget), it didn't fully converge. SLSQP then refines from DE's best point using gradients. In production, increase DE iterations to 300–500 for tight convergence.

The objective function Objective_2 is composed of fuel costs, steam utility costs, electricity costs, and DMW costs — all wired through the inferred tag dependency chain, so optimizing any setpoint automatically propagates through to the cost objective.

def run_bayesian_optimizer(
    opt_problem, current_data,
    max_iter=100, seed=42
):
    # Build objective function from problem dict
    obj_func, var_names, bounds = \
        _build_objective_func(opt_problem, current_data)

    # Differential evolution — global, non-convex
    result = differential_evolution(
        obj_func,
        bounds=bounds,          # from variables table
        maxiter=max_iter,
        seed=seed,
        mutation=(0.5, 1.0),
        recombination=0.7,
    )

    return result.x, result.fun, result.success

# SLSQP fallback with constraints:
result = minimize(
    obj_func, x0=initial_x,
    method="SLSQP",
    bounds=bounds_list,
    constraints=scipy_constraints,
    options={"maxiter": 500}
)

def compute_opportunity_tags(df_merged):
    # Find all tags that have BOTH _actual and _optimum
    actual_tags  = {c[:-7] for c in df
                    if c.endswith("_actual")}
    optimum_tags = {c[:-8] for c in df
                    if c.endswith("_optimum")}
    paired = actual_tags & optimum_tags

    for tag in paired:
        df[f"{tag}_opportunity"] = (
            df[f"{tag}_actual"] -
            df[f"{tag}_optimum"]
        )
    return df
